Retention model

Retention is project scoped.

Retention is project scoped. Changing one project's Data tab does not change another project's history. The fields are separated because logs and metrics, trace spans, and closed error groups often have different review value after the first few weeks.

Archives are not a full backup

Telemetry archives are for older event history. They do not preserve user accounts, project settings, API keys, memberships, or every workflow decision. Keep ordinary backups for full restore needs.
