Delivery path

Check the visible choices before debugging delivery.

PieceWhat to check
User preferenceThe right project member enabled the matching notification path for this project.
Project accessThe recipient is still a project owner or project member.
Alert conditionThe exact condition happened: a new group, reopened group, threshold crossing, assignment, digest schedule, or operational notice.
Email deliveryIf the UI choices look right but nothing arrives, verify ordinary account email first, then check the instance's email and background-processing setup.

Verify

Test from simplest mail to project-specific mail.

  1. Open the user's notification settings for the project.
  2. Confirm the matching path is enabled and not blocked by quiet hours or hourly caps.
  3. Create a new error group with a unique fingerprint.
  4. If the message does not arrive, test password reset email, then check instance-level email delivery logs.

For exact SMTP variables, read deployment email settings. For symptoms, read email troubleshooting.
