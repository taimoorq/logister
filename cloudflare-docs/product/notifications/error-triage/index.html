Settings

Each option answers a different question.

SettingUse it forDefault
New error groupsA new grouped issue was created for a new fingerprint.On
Reopened error groupsA resolved, ignored, or archived group received a new event.On
Lifetime count milestonesOne group's total lifetime occurrence count reached a notable number.Off
Single-error volume thresholdOne group received enough events inside the configured recent window.Off, 25 events in 60 minutes

Do not confuse the thresholds

Milestones look at one group's lifetime total. The single-error threshold looks at recent volume for one group. Project-wide spike alerts live on the Project health page.

Verify

Trigger the exact condition you enabled.

  1. For new group mail, send a test error with a unique fingerprint.
  2. For regression mail, close the group and send the same fingerprint again.
  3. For frequent-error mail, temporarily lower the count and window, then send repeated events for the same fingerprint.
