Notifications / Project health

Email me when the project itself looks unhealthy.

Use Project > Settings > Notifications > Project health alerts for project-wide volume, latency, and monitor problems.

Settings

These settings look across the project.

SettingUse it forDefault
Project-wide error spikeTotal project error events crossed the configured count inside the configured window.Off, 100 events in 15 minutes
Project p95 latencyProject p95 transaction duration over the last 15 minutes crossed the configured millisecond limit.Off, 1,000 ms
Check-in monitorsA check-in monitor was missed or later recovered.On

Project-wide error spike is not the same as the single-error volume threshold. This setting counts all project error events in the window.

Verify

Health alerts depend on sweeps.

Keep the worker and scheduled sweeps running. To test, temporarily lower the threshold, generate matching events, then watch the worker enqueue a project health notification.
